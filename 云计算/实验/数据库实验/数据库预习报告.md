# 实验四 预习报告：托管关系型数据库服务（PolarDB / RDS）

## 一、实验目的
这次实验是一次动手练习，目标很直接：在阿里云控制台上亲自创建数据库实例、建立访问账号、用内置的 SQL 编辑器跑几条 SQL，并学会看监控数据。通过操作你会理解为什么生产环境常把数据库放到云上，以及日常运维需要关心哪些点。

具体目标：

- 能说出 PolarDB 的典型使用场景；
- 会在控制台创建实例并配置访问账号；
- 能在控制台或客户端执行基本的 DDL/DML（建库、建表、增删改查）；
- 会查看并解读常见的性能指标（CPU、TPS、慢查询等）。

## 二、前置知识

- 关系型数据库基础概念：数据库、表、索引、事务与用户权限；
- 常用 SQL：CREATE、INSERT、SELECT（以及简单的 UPDATE/DELETE）；
- 基本的网络/安全概念：白名单（安全组）设置与访问控制。

## 三、实验准备

- 拥有可用的阿里云账号与实验资源（PolarDB 或 RDS 实例）；
- 准备一个能连数据库的客户端（控制台自带的 SQL 编辑器、MySQL Workbench、DBeaver 等均可）；
- 预备一组测试 SQL：创建库与表、插入若干测试数据、执行简单查询。

建议先把自己的公网 IP 加入白名单，以免连接被拒；实验结束后记得撤回或收紧白名单，避免长期暴露。

## 四、实验流程（建议按此顺序操作）

1. 登录阿里云控制台，打开 `PolarDB`（或 `RDS`）服务页面，创建或使用实验提供的实例。
2. 在实例管理界面进入“账号管理”或“数据库账户”，新建一个测试账号并分配必要权限（如创建表、插入、查询）。
3. 使用控制台内置的 SQL 编辑器（或本地客户端）连接数据库，执行示例 SQL：

	- 创建数据库：`CREATE DATABASE demo_db;`
	- 切换库并建表：`USE demo_db; CREATE TABLE users (...);`
	- 插入并查询数据：`INSERT ...; SELECT ...;`

4. 打开控制台的“监控/性能”页面，观察并记录关键指标：CPU 利用率、TPS/QPS、活动连接数、慢查询数等。
5. 根据监控信息判断是否存在性能问题（例如慢查询或连接数激增），并记录你的初步分析和处理建议。

## 五、预期结果

1. 成功创建并配置一个可用的 PolarDB 数据库账户；
2. 能在控制台的 SQL 编辑器中编写并执行建表、插入、查询的 SQL；
3. 能查看数据库实例的监控页面，读出并简单解释 CPU、TPS、慢查询等指标。
