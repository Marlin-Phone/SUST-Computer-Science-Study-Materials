# 实验二 预习报告 — 常见网络测试命令使用

## 一、实验目的
- 掌握 Ping、tracert、nslookup、arp、route、netstat、ipconfig 等常用命令的使用方法与用途。
- 学会用这些命令进行网络故障排查和调试。
- 熟悉远程桌面的基本启用与连接流程。

## 二、实验器材
- 一台装有 Windows 系统的计算机。

## 三、常用命令简介与用途
- `ping <IP或域名>`：测试主机连通性，测量往返时延与丢包率。常用于判断目标是否可达。
- `tracert <目标>`：跟踪数据包经过的路由路径，定位路由或链路故障的跳点。
- `nslookup <域名>`：查询域名到 IP 的解析信息（也可指定 DNS 服务器）。
- `arp -a`：查看本机 ARP 缓存表，显示 IP ↔ MAC 映射。
- `route print`（或 `route`）：查看/修改本机路由表，诊断路由配置问题。
- `netstat -ano`：显示当前 TCP/UDP 连接及监听端口、关联 PID，便于定位占用端口的进程。
- `ipconfig /all`：显示本机所有网络适配器的配置详情；`ipconfig /renew` 和 `ipconfig /release` 用于 DHCP 地址续约/释放。

## 四、远程桌面使用要点
1. 创建用户并设置密码：控制面板 → 用户账户 → 新建账户并设置密码。
2. 启用远程桌面：右键“此电脑”→“属性”→“远程设置”→选中“允许运行任意版本远程桌面的计算机连接（较不安全）”或相应选项。
3. 远程连接：启动“远程桌面连接”应用，输入目标主机 IP，使用用户名和密码登录。
